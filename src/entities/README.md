# Entities Layer

## Overview

ビジネスエンティティ（ドメインモデル）を定義する層です。アプリケーションの核心となるデータモデルと、それに関連するビジネスロジックを提供します。

## Responsibilities

- **データモデル定義**: エンティティの型定義とデータ構造
- **ビジネスロジック**: エンティティ固有の計算や変換ロジック
- **状態管理**: エンティティの状態を管理するストアやフック
- **UIコンポーネント**: エンティティを表示するための基本的なUIコンポーネント

## Structure

```
entities/
└── family/
    ├── model/
    │   ├── types.ts          # 型定義
    │   └── store.ts           # 状態管理
    ├── lib/
    │   └── age-calculator.ts # 年齢計算ロジック
    └── ui/
        └── AgeDisplay.tsx     # 年齢表示コンポーネント
```

## Rules

- `shared/`のみを参照可能（他のエンティティや上位層を参照してはいけない）
- エンティティ間の依存関係は避ける
- 純粋なビジネスロジックに集中
- フレームワーク固有の実装を避け、可能な限り純粋な関数として実装

## Example

`family`エンティティは、家族メンバーのデータモデル（名前、生年月日、役割）と、年齢計算などのビジネスロジック、そして状態管理機能を提供します。このエンティティは、アプリケーション全体で家族情報を扱う際の基盤となります。
