# Features Layer

## Overview

ユーザーが実行できる具体的な機能を定義する層です。ユーザーアクションに関連するUIとビジネスロジックを含みます。

## Responsibilities

- **ユーザーアクション**: ボタンクリック、フォーム送信などのユーザー操作に対応
- **機能固有のUI**: その機能に特化したUIコンポーネント
- **ビジネスロジック**: 機能を実現するためのロジック（バリデーション、データ変換など）
- **状態管理**: 機能固有の一時的な状態管理

## Structure

```
features/
└── edit-family-member/
    └── ui/
        └── FamilyMemberCard.tsx  # 家族メンバー編集カード
```

## Rules

- `entities/`と`shared/`を利用して機能を実装
- 他の`features/`を参照してもよいが、循環依存に注意
- 1つの機能は1つのディレクトリにまとめる
- 機能は独立して動作可能であるべき

## Example

`edit-family-member`機能は、家族メンバーの情報を編集するためのUIとバリデーションロジックを提供します。`family`エンティティのデータモデルを利用して、メンバー情報の編集を実現します。
